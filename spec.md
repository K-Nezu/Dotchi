# 新サービス名

# アプリ要件定義書：Dotchi（仮称）

## 1. コンセプト

- **コア価値:** **「今この瞬間」の迷いを、5分間だけ世界に放流して、直感的に解決する。**
- **キーワード:** 刹那感（5分）、ブラインド投票（後出し開票）、直感、シンクロ、ポイ活。
- **デザイン指針:** 「勝負」ではなく「心地よさ」。パステルカラーや丸みを帯びたミニマルなUI。

## 2. ユーザー体験（UX）

- **投稿者:** 迷いを書き出した瞬間に「あとは任せた」という解放感を得る。
- **回答者:** 5分以内にポチッと助ける「小さな善意」と、多数派を当てる「クイズの快感」を得る。

## 3. 主要機能（MVP）

## ① 投稿機能（マルチモード）

- **画像2択モード:** 写真2枚をアップロードして比較。
- **テキスト2択モード:** 文字（各50文字以内）のみの比較。
    - テキストモード時は、背景にグラデーションを施した「カードUI」を自動生成。
- **共通制限:** 全ての投稿は**「5分間」**で終了し、トップのタイムラインから自動消滅する。

## ② ブラインド投票機能

- **非公開制:** 5分が経過するまで、どちらが優勢か（現在の票数）は一切表示しない。
- **ステータス表示:** 具体的な数字は出さず、「〇〇人が検討中...」「誰かが投票しました」という気配だけをアニメーションで見せる。
- **操作感:** 選択した瞬間に心地よいフィードバック（振動やエフェクト）を返す。

## ③ 5分後の「開票」演出

- **結果解禁:** 5分経過後、リロードなしで自動的に結果画面へ切り替わる（または通知で誘導）。
- **アニメーション:** ゲージが伸びて勝敗が決まる、キラキラとした達成感のある演出。
- **インセンティブ:** 多数派を選んだユーザーに「擬似ポイント（スコア）」を付与。

## ④ 運用・コスト最適化

- **AIブースト:** 投稿直後にAIエージェントが数票を投じ、無回答（0票）を防止。
- **データクリーンアップ:** 5分経過した不要な画像データ・DBレコードを自動削除し、ストレージコストを極小化。

## 4. 技術スタック（推奨）

- **Frontend:** Next.js (App Router), Tailwind CSS
- **Backend:** Supabase (Auth, DB, Storage, Edge Functions)
- **Infrastructure:** Vercel (Hobby Plan)

## 5. UI/UX詳細ルール

- **カウントダウン:** 巨大なタイマーではなく、進捗バーや控えめな数字で「残り時間」を表示。
- **ログイン:** 最初はログイン不要（匿名）で投票可能にし、離脱を防ぐ。
