# Dotchi 要件定義書

## 1. コンセプト

- **一言説明:** 「これが一番でしょ」って出し合って、5分で決まるアプリ
- **キャッチコピー:** 「一番、どっち？」
- **プロダクト定義:** エンタメの「一番」を5分で決めるマッチアプリ
- **キーワード:** 一番、マッチ、5分、カジュアル、エンタメ
- **思想:** 「自分の一番を掲げて、世間に問う」「軽いけど本気」
- **体験モデル:** スポーツ観戦。プレイヤー（投稿者）が戦い、オーディエンス（投票者）がコメント付きで応援する。蓄積されたスコアが「みんなの評価」として残る
- **トーン:** カジュアル。煽りや対決感は出さない。問いかけの温度

---

## 2. コアループ

### 投稿者（チャレンジャー募集）
1. ジャンル（音楽/映画/ゲーム等）を選ぶ
2. API検索から「自分の一番」を選択
3. 「これより上ある？」的なキャプション（任意）をつけて投稿
4. タイムラインに**片側だけ埋まった状態**で表示される

### チャレンジャー（マッチ成立）
1. タイムラインで「空き枠」の投稿を見つける
2. 「いや、こっちでしょ」とAPI検索から対抗を選んで投稿
3. **マッチ成立** → 5分間の投票が開始

### 投票者
1. マッチ中の投稿を見て、どっちが「一番」かタップで投票
2. 音楽なら試聴してから投票できる

### 決着
1. 5分経過 → 投票締切
2. 多数派が「一番」として認定される

---

## 3. 対応ジャンル

MVPは音楽のみ。検証後に順次拡張。

| ジャンル | API | MVP | 備考 |
|---------|-----|-----|------|
| 音楽 | iTunes Search API | Yes | 30秒試聴あり。実装済み |
| 映画 | TMDB API | 後日 | ポスター画像+あらすじ |
| ゲーム | IGDB API | 後日 | カバーアート+メタデータ |
| アニメ | 未定 | 後日 | |

### 投稿方式
- 全ジャンル共通で**API検索→選択**方式
- ユーザーが画像を撮影・アップロードする必要がない
- 公式アートワークで見た目が常にリッチ
- ストレージ負荷はURLのみ（Supabase Storage不要）

## 4. 投稿の状態遷移

```
[募集中] → [マッチ成立・投票中] → [決着・結果表示(1分)] → [再募集] → ... → [殿堂入り]
```

| 状態 | 説明 | 表示 |
|------|------|------|
| 募集中 | 片側のみ投稿済み。チャレンジャー待ち | 片側コンテンツ + 空き枠 |
| 投票中 | マッチ成立。5分間のカウントダウン中 | 両側コンテンツ + 投票可能 |
| 集計中 | 投票済みユーザーの待機状態 | オーバーレイ + 呼吸エフェクト |
| 決着 | 5分経過。結果表示（1分間） | パーセンテージ + 勝者強調 |
| 再募集 | 勝者が残り、空き枠が再オープン | 勝者コンテンツ + 空き枠 + 戦績表示 |
| 殿堂入り | 5勝達成 | 殿堂入りバッジ + 最終戦績 |
| 敗退 | マッチに負けた投稿 | タイムラインから非表示（履歴には残る） |

### フロー詳細
1. 投稿者が「自分の一番」を掲げる → 左側に表示、右側は空き枠
2. 閲覧者が空き枠に対抗コンテンツを投稿 → マッチ成立
3. 5分間の投票
4. 決着 → 結果表示（1分間）
5. 勝者が残り、右側が再び空き枠に（戦績「1勝」表示）
6. 新たなチャレンジャーがマッチ → 2回目の投票...
7. 5勝達成 → 殿堂入り認定

### 殿堂入りシステム
- **条件:** 連続5勝
- 途中で負けたらその投稿は終了（敗退）
- 殿堂入りしたコンテンツは専用ページに掲載
- 「殿堂入り」はシェアしたくなる称号として機能する
- DBは1対1の構造のまま。勝者投稿を再利用するだけ

### 募集中投稿のTTL
- マッチ未成立のまま24時間経過 → 自動削除

## 5. 主要機能

### 投稿機能（一番を掲げる）
- ジャンル選択 → API検索 → コンテンツ選択 → キャプション（任意） → 投稿
- 投稿は「片側だけ」の状態でタイムラインに表示

### マッチ機能（挑む）
- タイムラインの空き枠をタップ → API検索 → 対抗コンテンツ選択 → マッチ成立
- マッチ成立と同時に5分タイマー開始

### 投票機能
- ログイン不要・匿名（デバイスIDベース）
- タップで投票（即フィードバック）
- 音楽モードでは再生ボタンで試聴してから投票
- 1投稿につき1票
- 投票時に一言コメント（任意、最大100文字）を添えられる

### リアルタイムコメント（観客席）
- 投票者が添えたコメントがリアルタイムでPostCard下部に表示
- どちら側に投票したかラベル付き（A派/B派が可視化される）
- Supabase Realtimeで即反映。スポーツ観戦のような臨場感
- コメントはスクロール可能なフィードとして積み上がる

### 結果表示
- 5分経過後に自動表示（カウントダウン演出あり）
- パーセンテージをアニメーション表示
- 勝者側を強調、敗者側はグレースケール+透過

### シェア機能
- マッチ中：「どっちが一番？」でシェア（X / LINE / Web Share API）
- 決着後：結果をシェア
- 投稿ごとにユニークURL（`/posts/[id]`）

### 時間制御
- マッチ成立から5分で投票締切
- 募集中の投稿は24時間でマッチ未成立なら自動削除
- 決着後30分でタイムラインから非表示

## 6. UI/UX設計原則

### 基本思想
- 軽い。カジュアル。でも手を抜いてない
- Apple風のミニマルデザイン
- 「勝負」ではなく「問いかけ」のトーン

### デザインルール
- **レイアウト:** フィード型。カードベース
- **マッチ表示:** 左右に2つのコンテンツ。未マッチ時は片側が空き枠
- **空き枠:** タップを誘うデザイン（破線ボーダー等）。「挑む」ではなく「選ぶ」的なラベル
- **色:** モノトーン基調。foreground/muted/borderの3階層。アクセントカラーは使わない
- **角丸:** カード・ボタンは`rounded-xl`〜`rounded-full`
- **影:** 使わない
- **余白:** 適度に空気感を持たせる
- **絵文字:** UIのラベルやボタンには使わない
- **タイポグラフィ:** 曲名/タイトル`text-sm font-semibold`、アーティスト名等`text-xs text-muted`
- **アニメーション:** 控えめ。投票時のフィードバック、結果表示のイージング

### 状態別UI

| 状態 | 表示 |
|------|------|
| 募集中 | コンテンツ + 空き枠（タップ誘導） |
| 投票中 | 両側コンテンツ + タイマー + 投票ボタン |
| 集計中（投票済） | オーバーレイ + 呼吸エフェクト + 残り時間 |
| 決着 | パーセンテージ + 勝者強調 |

### 賑わい感の演出（初期重要）
- リアルタイムの投票数アニメーション（票が入るたびにバーが動く）
- 「投票中」のカウントダウンが常に動いている
- 直近の決着結果をフィードに流す
- 初期は表示投稿数を絞り、「募集中で放置」の過疎感を出さない

### 禁止事項
- ログイン必須の導線
- 対決・バトル・煽り系の文言
- 過剰なアニメーション
- 重い影やグラデーション
- 説明的な文言

## 7. 匿名システム

- 投稿者・チャレンジャーはデバイスIDで識別（ログイン不要）
- 投稿ごとにランダムな匿名名とアバターを生成
- 将来的にアカウント機能追加の余地は残す

## 8. 技術スタック

| レイヤー | 技術 |
|---------|------|
| Frontend | Next.js 16（App Router）+ TypeScript + Tailwind CSS v4 |
| Backend | Supabase（Auth / DB / Realtime） |
| Hosting | Vercel（Hobby Plan） |
| 音楽API | iTunes Search API（JP） |

### 設計方針
- Auth不要（デバイスIDベース）
- モバイルファーストWeb（最大幅`max-w-xl`）
- PWA対応（ホーム画面追加可能）
- URLで即共有（LINE / Xで拡散）
- Supabase Storageは極力使わない（API URL保存で代替）

## 9. お題（Daily Theme）システム

### コンセプト
- 運営が毎日「お題」をSNS（X等）で発信し、ユーザーの投稿を促す
- お題はテーマだけ提示し、2択の選択肢はユーザーが自分で作る
- 同じお題に対して人によって違う2択が出てくること自体がコンテンツ

### 構造
- `daily_themes` テーブル: date(日付), title(お題), description(説明), examples(例の配列), mode(推奨ジャンル)
- 投稿の `theme_id` でお題と紐づけ（任意）
- タイムライン上部に `DailyThemeBanner` として表示
- 「このお題で投稿する」→ BottomSheetで投稿フォームが開く（ジャンルが自動選択される）

### 運用フロー
1. 運営がSupabaseに翌日のお題を登録（title, description, examples, mode）
2. 当日のSNS投稿で「今日のお題: ○○」と投げかける
3. ユーザーがアプリを開くとバナーにお題が表示される
4. ユーザーが自分の2択を作って投稿 → theme_idで紐づく

### お題の例
- 「2025年のゲームオブザイヤーは？」（mode: game）
- 「夏に聴きたい1曲は？」（mode: music）
- 「人生で一番泣いた映画は？」（mode: movie）
