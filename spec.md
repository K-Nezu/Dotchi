# Dotchi 要件定義書

## 1. コンセプト

- **コア価値:** 「今この瞬間」の迷いを、5分間だけ世界に放流し、意思決定を外注する
- **プロダクト定義:** 人生の瞬間判断OS / 意思決定の外注インフラ
- **キーワード:** 刹那感（5分）、リアルタイム性、直感、匿名、軽い善意
- **思想:** 「考える前に投げる」「繋がりすぎないが、影響は受ける」

---

## 2. 対応メディア

画像・テキスト・音楽の2択コンテンツを投稿できる。将来的に動画にも対応予定。

| モード | 選択肢の構成 | 備考 |
|--------|-------------|------|
| 画像 | 写真2枚 | 質問テキストは任意 |
| テキスト | テキスト2択 | 質問テキストは任意 |
| 音楽 | 楽曲2曲 | iTunes APIで検索。試聴あり |

---

## 3. ユーザー体験（UX）

### 投稿者
- 迷った瞬間に即投稿（画像なら撮影→2枚選ぶだけ）
- 投稿後すぐに「委ねた感」「解放感」を得る
- 5分後、世界の答えを知る

### 回答者
- タイムラインで「今この人が迷っている」リアルタイム性を感じる
- ワンタップで意思決定に関与する
- 音楽モードでは試聴してから投票できる

### コア体験
- 投稿：思考前に投げる
- 投票：反射で選ぶ（音楽は聴いて選ぶ）
- 結果：自分の影響を知る

---

## 4. 主要機能

### 投稿機能
- ボトムシートUIで素早く作成
- 画像モード：端末から写真2枚を選択（リサイズ・圧縮してSupabase Storageへアップロード）
- テキストモード：選択肢A/Bを入力
- 音楽モード：iTunes APIで楽曲検索→2曲選択
- 質問テキストは全モード共通で任意入力
- 投稿後即座にタイマー開始（5分間）

### 投票機能
- ログイン不要・匿名（デバイスIDベース）
- タップ時に即フィードバック（リップルエフェクト + バイブレーション）
- 音楽モードでは再生ボタンで試聴してから投票
- 投票後は「集計中」状態で結果を待つ

### 結果表示
- 5分経過後に自動表示（カウントダウン演出あり）
- パーセンテージをアニメーション表示
- 勝者側を強調、敗者側はグレースケール+透過
- バッジ表示：「多数派」「あなた」「本人」

### 投稿者の選択（ポスターチョイス）
- 結果表示後、投稿者自身に「で、あなたはどっちにした？」と問う
- 投稿者の最終選択も結果に表示される

### シェア機能
- 投票中：LINEで友達に聞いてもらう（リンク型テキストボタン）
- 結果後：結果をシェア（X / LINE / Web Share API）
- 投稿ごとにユニークURL（`/posts/[id]`）

### 時間制御
- 投稿から5分で投票締切
- 締切後30分でタイムラインから非表示
- 残り3秒でカウントダウン演出→フラッシュ→結果表示

## 5. UI/UX設計原則

### 基本思想
- 思考させない / 迷わせない / 即行動させる
- Apple風のミニマルデザイン

### デザインルール
- **レイアウト:** Instagram型フィード。画像は全幅（端から端まで）で並べ、画像自体がコンテンツの区切りになる
- **2択表示:** `grid-cols-2 gap-[2px]`で2枚並列。隙間は最小限の白線のみ
- **投稿間の区切り:** 薄い罫線（`divide-y divide-border/60`）
- **色:** モノトーン基調。foreground/muted/borderの3階層。アクセントカラーは使わない
- **角丸:** 画像部分はフラット（全幅）。ボタン類は`rounded-full`
- **影:** 使わない
- **ボーダー:** 極力使わない。使う場合は`border-border`程度
- **余白:** ヘッダー・フッター部分は`px-4 py-3`。画像部分はパディングなし
- **絵文字:** UIのラベルやボタンには使わない。匿名アバターのみ例外
- **タイポグラフィ:** 名前`text-sm font-medium`、質問`text-[15px] font-semibold`、補助テキスト`text-xs text-muted`
- **アニメーション:** 控えめ。待機中は呼吸エフェクト（オーバーレイの明滅）、結果はイージングアニメーション

### 状態別UI

| 状態 | 表示 |
|------|------|
| 投票受付中 | 2択タップ可能。音楽は再生ボタン付き |
| 待機中（投票済/投稿者） | オーバーレイ + 呼吸エフェクト + 「?」 |
| カウントダウン | 中央に数字（3→2→1） |
| 結果表示 | パーセンテージ + 勝者強調 |

### 禁止事項
- ログイン必須の導線
- 複雑な入力フォーム
- 過剰なアニメーション（ネオンボーダー等の装飾的演出）
- 重い影やグラデーション
- 説明的な文言

---

## 6. 匿名システム

- 投稿者はデバイスIDで識別（ログイン不要）
- 投稿ごとにランダムな匿名名（「匿名のキツネ」等）とアバター絵文字を生成
- 投稿者と回答者は同じ投稿に対して役割が分離される

---

## 7. 技術スタック

| レイヤー | 技術 |
|---------|------|
| Frontend | Next.js 16（App Router）+ TypeScript + Tailwind CSS v4 |
| Backend | Supabase（Auth / DB / Storage / Realtime） |
| Hosting | Vercel（Hobby Plan） |
| 音楽API | iTunes Search API（JP） |
| 画像処理 | クライアントサイドでリサイズ・WebP圧縮 |

### 設計方針
- Auth不要（デバイスIDベース）
- モバイルファーストWeb（最大幅`max-w-xl`）
- PWA対応（ホーム画面追加可能）
- URLで即共有（LINE / Xで拡散）

- 投票数：50件/日
- 投稿→投票完了率
- シェア経由の流入率

---

## 10. 本質

このプロダクトは「人の意思決定を外に委ねる文化」を作る。

そのために必要なのは：
- 投稿の速さ（思考前に投げる）
- 投票の軽さ（反射で選ぶ）
- 関与感（影響を実感する）

一言でいうと：**考える前に投げて、世界に決めてもらう**
